# 批发商城开工包-权限码矩阵

## 1. 说明

本文档用于补齐批发商城项目正式开工前的权限设计基线。

- 适用范围：后端接口、管理端菜单、代理商页面、供货商页面
- 对齐工程：`manager-api`、`buyer-api`、`seller-api`
- 对齐风格：沿用现有项目 `模块:资源:动作` 的权限命名思路
- 角色侧重点：
  - 平台后台：治理、审核、配置、统计
  - 供货商：商品、订单、发货、营销、资产
  - 代理商：店铺、批发、订单、核销、对账、经营概览
  - 消费者：浏览、下单、拼团、积分、售后

## 2. 角色编码建议

| 角色名 | roleCode | 说明 |
|---|---|---|
| 平台后台 | `ROLE_PLATFORM` | 管理端治理角色 |
| 供货商 | `ROLE_SUPPLIER` | 店铺供货经营角色 |
| 代理商 | `ROLE_AGENT` | 区域代理经营角色 |
| 消费者 | `ROLE_CONSUMER` | 前台普通用户 |

## 8. 实施要求

- 后续新增菜单时，菜单编码、权限码、接口路径必须三者对应
- 管理端菜单树、角色菜单、接口路径需同步录入
- 代理商与消费者同属 `buyer-api`，必须依赖角色编码做二次数据隔离
- 供货商权限继续接入现有 `seller-api` 权限缓存机制
- 中文菜单名、中文接口说明、中文注释必须与本矩阵一致
