# 批发商城开工包-状态枚举与流转

## 1. 说明

本文档用于补齐批发商城正式开发前的状态设计基线。

- 目标：统一页面状态、接口状态、表字段状态
- 要求：先定义枚举，再落库字段，再写接口流转
- 约束：所有状态字段必须带中文备注，不允许只保留数字码值含义不明

## 2. 通用状态字段约定

| 字段名 | 说明 |
|---|---|
| `status` | 通用启停状态 |
| `audit_status` | 审核状态 |
| `order_status` | 订单状态 |
| `order_type` | 订单类型 |
| `pintuan_status` | 拼团状态 |
| `verify_status` | 核销状态 |
| `settlement_status` | 结算状态 |
| `reconciliation_status` | 对账状态 |
| `bind_status` | 绑定状态 |

## 4. 状态流转清单

### 4.1 店铺入驻申请流转

| 当前状态 | 触发动作 | 下一状态 | 前置条件 |
|---|---|---|---|
| `DRAFT` | 提交申请 | `SUBMITTED` | 主体资料完整 |
| `SUBMITTED` | 审核通过 | `APPROVED` | 平台审核通过 |
| `SUBMITTED` | 审核驳回 | `REJECTED` | 平台填写驳回原因 |
| `REJECTED` | 重新提交 | `SUBMITTED` | 补齐驳回项 |
| `APPROVED` | 平台冻结 | `FROZEN` | 存在违规或经营异常 |
| `FROZEN` | 解冻 | `APPROVED` | 平台复核通过 |

### 4.2 订单主流程

| 当前状态 | 触发动作 | 下一状态 | 前置条件 |
|---|---|---|---|
| `UNPAID` | 支付成功 | `PAID` | 支付回调成功 |
| `UNPAID` | 超时取消 | `CANCELLED` | 超过支付时限 |
| `PAID` | 商家发货 | `DELIVERED` | 非核销订单 |
| `PAID` | 生成核销码 | `VERIFICATION_PENDING` | 核销类订单 |
| `DELIVERED` | 用户收货 | `RECEIVED` | 物流已发出 |
| `RECEIVED` | 评价完成/自动完成 | `COMPLETED` | 到达确认收货后置规则 |
| `VERIFICATION_PENDING` | 门店核销 | `COMPLETED` | 核销码校验成功 |
| `PAID` | 发起售后 | `AFTER_SALE` | 满足售后时效 |
| `DELIVERED` | 发起售后 | `AFTER_SALE` | 满足售后时效 |

### 4.3 拼团流程

| 当前状态 | 触发动作 | 下一状态 | 前置条件 |
|---|---|---|---|
| `NOT_STARTED` | 到开始时间 | `ONGOING` | 活动启用 |
| `ONGOING` | 达到成团人数 | `SUCCESS` | 有效参团人数满足要求 |
| `ONGOING` | 到结束时间未成团 | `FAILED` | 未达到成团人数 |
| `SUCCESS` | 到活动结束 | `ENDED` | 活动自然结束 |
| `NOT_STARTED` | 平台关闭 | `CLOSED` | 人工关闭 |
| `ONGOING` | 平台关闭 | `CLOSED` | 人工关闭 |

补充规则：

- `FAILED` 拼团订单需自动退款或原路退回积分/余额
- `SUCCESS` 后对应订单才允许进入发货或核销履约

### 4.4 核销流程

| 当前状态 | 触发动作 | 下一状态 | 前置条件 |
|---|---|---|---|
| `NOT_GENERATED` | 支付成功生成核销码 | `PENDING` | 核销类商品 |
| `PENDING` | 门店核销成功 | `VERIFIED` | 核销码正确且未过期 |
| `PENDING` | 超过有效期 | `EXPIRED` | 超过核销有效期 |
| `PENDING` | 订单取消/退款 | `CANCELLED` | 订单终止 |

### 4.5 分账流程

| 当前状态 | 触发动作 | 下一状态 | 前置条件 |
|---|---|---|---|
| `WAIT_CALCULATE` | 结算任务计算 | `WAIT_AUDIT` | 有可分账订单 |
| `WAIT_AUDIT` | 平台审核通过 | `WAIT_SETTLEMENT` | 审核通过 |
| `WAIT_AUDIT` | 平台审核驳回 | `AUDIT_REJECTED` | 填写驳回原因 |
| `AUDIT_REJECTED` | 重新发起 | `WAIT_AUDIT` | 修正数据 |
| `WAIT_SETTLEMENT` | 结算成功 | `SETTLED` | 钱包入账成功 |
| `WAIT_SETTLEMENT` | 结算失败 | `FAILED` | 三方结算失败 |

### 4.6 对账流程

| 当前状态 | 触发动作 | 下一状态 | 前置条件 |
|---|---|---|---|
| `INIT` | 生成对账单 | `PENDING_CONFIRM` | 对账周期结束 |
| `PENDING_CONFIRM` | 代理商确认 | `CONFIRMED` | 金额核对无误 |
| `PENDING_CONFIRM` | 代理商发起异议 | `DISPUTED` | 存在金额差异 |
| `DISPUTED` | 平台处理完成 | `CONFIRMED` | 异议处理完成 |
| `CONFIRMED` | 归档完成 | `FINISHED` | 周期关闭 |

## 5. 开发要求

- Java 枚举类必须写中文注释
- Swagger 对状态字段必须标注枚举说明
- 表结构备注必须写明状态枚举含义
- 状态变更接口必须校验当前状态，不允许跨状态直接跳转
- 审核、核销、结算、退款类接口必须记录操作人、操作时间、操作备注
