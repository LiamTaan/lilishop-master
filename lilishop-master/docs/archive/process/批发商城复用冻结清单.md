# 批发商城复用冻结清单

## 1. 说明

本清单作为“里程碑0”正式冻结产物，约束批发商城阶段一开发不得重复造轮子。

- 结论类型固定为：
  - `直接复用`
  - `扩字段复用`
  - `扩接口复用`
  - `新建`
- 本次先覆盖“里程碑0+1”直接涉及的模块：店铺、权限、会员、区域、商品、订单、钱包、核销、统计、采购与基础设施。

## 2. 基础设施复用结论

| 模块 | 现状 | 结论 | 说明 |
|---|---|---|---|
| `modules/payment` | 已有支付、退款、钱包支付、回调链路 | 直接复用 | 阶段一不改支付主链 |
| `modules/logistics` | 已有物流查询、面单、地址能力 | 直接复用 | 阶段一仅复用，不扩展 |
| `modules/sms` | 已有短信发送与校验 | 直接复用 | 登录、通知继续走现有链路 |
| `modules/file` | 已有 OSS/文件上传 | 直接复用 | 入驻资料图片沿用现有上传能力 |
| `modules/im` + `im-api` | 已有 IM 能力 | 直接复用 | 阶段一不新增 IM 逻辑 |
| `modules/search` | 已有商品索引同步 | 直接复用 | 店铺字段更新继续走现有 MQ + ES 同步 |
| RocketMQ / Quartz / xxl-job | 已有 | 直接复用 | 阶段一不新建平行消息或调度机制 |

## 3. 业务模块复用结论

| 模块 | 现状 | 结论 | 说明 |
|---|---|---|---|
| `modules/store` | 已有店铺申请、审核、详情、启停 | 扩字段复用 | 扩主体类型、审核状态、审核备注与申请资料字段 |
| `modules/member` | 已有会员体系 | 扩接口复用 | 代理商作为会员扩展身份，不新建登录体系 |
| `modules/permission` | 已有后台角色、菜单、用户角色 | 扩接口复用 | 继续复用后台菜单/角色能力，补代理商治理接口与权限码 |
| `modules/system` | 已有区域表 `li_region` | 直接复用 | 代理商区域直接复用区域数据 |
| `modules/goods` | 已有商品、批发阶梯价 | 直接复用 | 阶段一只复用，不改商品模型 |
| `modules/order` | 已有订单主链 | 直接复用 | 阶段一不改订单主链 |
| `modules/wallet` | 已有钱包、提现 | 直接复用 | 阶段一不改资金闭环 |
| `modules/verification` | 已有核销能力 | 直接复用 | 阶段一不改核销链 |
| `modules/statistics` | 已有统计底座 | 直接复用 | 阶段一不落统计新接口 |

## 4. 表结构冻结结论

| 对象 | 结论 | 说明 |
|---|---|---|
| `li_store` | 扩字段复用 | 新增 `apply_type`、`store_type`、`audit_status`、`audit_remark` |
| `li_store_detail` | 扩字段复用 | 新增主体差异资料字段与证照字段 |
| `li_member` | 直接复用 | 代理商通过关系表绑定，不直接改会员主表 |
| `li_role` / `li_user_role` / `li_menu` / `li_role_menu` | 直接复用 | 权限矩阵通过初始化数据接入 |
| `agent_role_relation` | 新建 | 承载代理商身份、区域、状态 |
| `agent_store_bind` | 新建 | 承载代理商与店铺绑定、审核、解绑 |

## 5. 接口冻结结论

| 接口域 | 结论 | 说明 |
|---|---|---|
| `/manager/store/store` | 扩接口复用 | 补新的审核入参模型，支持通过/驳回/冻结 |
| `/buyer/store/store/apply` | 扩接口复用 | 继续复用当前店铺申请主链，扩申请字段 |
| `/manager/agent/*` | 新建 | 平台代理商身份与绑定治理接口 |
| `/buyer/agent/*` | 新建 | 当前会员代理商身份校验、绑定店铺查询、申请单查询 |

## 6. 禁止重写清单

- 禁止重写支付回调、退款、钱包支付主链
- 禁止新造会员登录态或 token 角色体系
- 禁止新造区域/行政区数据表
- 禁止为了代理商身份重建店铺主表
- 禁止新造后台菜单/角色平行体系

## 7. 本轮实施边界

- 本轮只交付里程碑0+1：复用冻结 + 代理商/店铺/权限/入驻审核主链
- 不同时推进商品治理、订单履约、营销、资金对账、统计大盘
- 管理端前端工程本轮只保留接口承接边界，不在本仓库内展开页面实现
