# 批发商城管理端前端工程方案

## 1. 方案来源

本方案直接承接《批发商城项目完整落地计划》中的平台管理端接入要求：

- 平台管理端新建独立前端工程，例如 `admin-web`
- 使用成熟 Vue3 Admin 开源后台模板作为底座快速接入
- 技术栈固定为 `Vue3 + TypeScript`
- UI 组件固定优先 `Element Plus`
- 独立工程，不嵌入当前 Java 多模块仓库
- 只做平台管理端，不新建代理商端、供货商端前端工程

因此，管理端前端建设的前提不是“工作区里已经有人放了 admin-web”，而是“本项目应主动新建独立管理端工程，并以成熟 Vue3 Admin 模板为底座落地”。

## 2. 建设目标

管理端前端工程目标不是做一个演示后台，而是直接承接批发商城平台治理页面，并与现有 `manager-api` 完整联调：

- 登录、Token、权限、动态菜单可用
- 核心治理菜单可访问
- 列表、详情、审核、状态操作、导出能力可统一承接
- 页面命名、字段、权限码、接口路径与后端文档一致

## 3. 底座选择原则

管理端底座必须满足以下要求：

- 基于 Vue3
- 使用 TypeScript
- 使用 Element Plus
- 支持动态路由
- 支持权限控制
- 支持可扩展菜单系统
- 支持通用表格、查询表单、详情弹窗、标签页等后台治理场景

本项目不要求在本轮锁死某一个具体开源仓库名称，但要求底座属于“成熟 Vue3 Admin 开源后台模板”，不能从空白脚手架手写一套后台骨架。

## 4. 工程定位

建议前端工程名称：

- `admin-web`

建议位置：

- 与 `lilishop-master` 平级，作为独立目录

说明：

- 后端仓库继续保留 Java 多模块结构
- 管理端前端独立维护构建、依赖、环境变量、打包流程
- 不将 Vue 工程直接塞入 `manager-api` 或 `framework`

## 5. 推荐目录结构

建议 `admin-web` 采用以下结构：

- `src/api`
  - 按业务域拆分接口文件
- `src/router`
  - 静态路由
  - 动态路由装配
- `src/store`
  - 用户、权限、字典、标签页
- `src/views`
  - 店铺治理
  - 商品治理
  - 订单治理
  - 营销治理
  - 资金治理
  - 数据概览
- `src/components`
  - 查询表单
  - 列表表格
  - 详情抽屉
  - 审核弹窗
  - 状态标签
- `src/utils`
  - Token
  - 权限判断
  - 枚举映射
  - 金额格式化
  - 时间格式化
- `src/styles`
  - 全局变量
  - 页面通用样式

## 8. 页面命名口径

管理端前端页面必须使用以下稳定名称：

- `首页分类配置`
  - 当前技术接口名仍为 `shortcutNav`
- `平台工作台`
  - 对应 `/manager/dashboard/wholesale`
- `平台经营概览`
  - 对应 `/manager/statistics/index`
- `分账规则`
  - 对应 `/manager/profitsharing/rule`
- `分账明细`
  - 对应 `/manager/profitsharing/record`
- `分账治理概览`
  - 对应 `/manager/profitsharing/balance`
- `采购对账`
  - 管理端采购账单治理页
- `资金对账`
  - 管理端资金账单治理页

禁止继续混用：

- 金刚区配置
- 首页快捷入口配置
- 数据概览 / 工作台 / 经营概览混指同页
- 分账管理 / 分账记录 / 平台对账中心混名

## 9. 权限接入原则

前端页面与按钮必须直接绑定后端权限体系：

- 菜单权限
- 路由权限
- 按钮权限

当前业务权限重点包括：

- `store:apply:query`
- `store:apply:audit`
- `store:manage:query`
- `store:manage:view`
- `store:manage:audit`
- `store:agent:bind`
- `goods:manage:query`
- `goods:manage:audit`
- `goods:manage:edit`
- `goods:category:query`
- `goods:category:edit`
- `operation:shortcutNav:edit`
- `promotion:points:query`
- `promotion:cardCoupon:query`
- `order:manage:query`
- `order:manage:view`
- `afterSale:manage:query`
- `afterSale:manage:audit`
- `verification:manage:query`
- `verification:manage:stats`
- `promotion:pintuan:query`
- `promotion:pintuan:edit`
- `profitsharing:rule:query`
- `profitsharing:rule:edit`
- `profitsharing:record:query`
- `profitsharing:settlement:settleAudit`
- `reconciliation:purchase:query`
- `reconciliation:fund:query`
- `dashboard:platform:view`
- `dashboard:platform:stats`

说明：

- `operation:shortcutNav:edit` 当前仍是技术权限名，前端页面展示语义按“首页分类配置”解释。

## 10. 联调要求

- 所有列表页统一分页、筛选、排序
- 所有详情页必须覆盖冻结文档字段
- 所有审核页必须展示备注和历史
- 所有状态按钮必须调用真实后端动作接口
- 所有状态文案来自后端枚举映射
- 不允许前端本地伪造审核流转

## 11. 当前结论

- 管理端前端不是等待外部仓库被放进工作区后再做。
- 按冻结计划，本项目就应该主动新建 `admin-web` 独立工程。
- 工程底座应采用成熟 `Vue3 Admin` 开源后台模板。
- 当前后端、权限、菜单、命名口径已基本冻结，可直接进入前端工程初始化阶段。
