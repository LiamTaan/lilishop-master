# 批发商城里程碑1接口权限矩阵

## 1. 说明

本文件用于约束“里程碑1”后端联调范围内的接口、菜单与权限初始化保持一致。

- 适用范围：
  - 平台管理端 `manager-api`
  - 买家端 `buyer-api`
- 本轮目标：
  - 后端接口先稳定
  - 菜单权限与接口前缀对齐
  - 为后续 `admin-web` 接入提供固定契约

## 2. 管理端菜单与接口矩阵

| 菜单标题 | 菜单路径 | 权限表达式 | 对应接口 |
|---|---|---|---|
| 代理商治理 | `agent-manage` | `/manager/agent/role*,/manager/agent/store*` | `POST /manager/agent/role`<br>`GET /manager/agent/role`<br>`POST /manager/agent/store`<br>`GET /manager/agent/store`<br>`PUT /manager/agent/store/{bindId}/audit`<br>`PUT /manager/agent/store/{bindId}/unbind` |
| 店铺审核历史 | `store-audit-log` | `/manager/store/store/audit/log*` | `GET /manager/store/store/audit/log/{storeId}` |

## 3. 管理端复用接口扩展

| 接口 | 说明 | 当前状态 |
|---|---|---|
| `PUT /manager/store/store/audit/{id}` | 新审核入参 `StoreAuditDTO`，支持通过、驳回、冻结、解冻备注流转 | 已落地 |
| `PUT /manager/store/store/audit/{id}/{passed}` | 兼容历史审核接口 | 保留兼容 |
| `GET /manager/store/store/get/detail/{storeId}` | 查看店铺详情及扩展申请资料 | 复用 |

## 4. 买家端代理商相关接口矩阵

| 接口前缀 | 说明 | 数据隔离规则 |
|---|---|---|
| `/buyer/agent/*` | 代理商身份判断、已绑定店铺查询 | 当前会员必须登录；代理商身份来自 `agent_role_relation` |
| `/buyer/order/order*` | 订单列表、详情、动作 | 若当前会员是代理商，仅可访问已审核通过绑定店铺的数据 |
| `/buyer/order/afterSale*` | 售后列表、详情、申请、取消、回寄、售后地址 | 若当前会员是代理商，仅可访问已审核通过绑定店铺的数据 |
| `/buyer/order/complain*` | 投诉详情、创建、沟通、取消 | 若当前会员是代理商，仅可访问已审核通过绑定店铺的数据 |
| `/buyer/trade/receipt*` | 发票列表、详情、开票、保存 | 强制当前会员归属；代理商额外受绑定店铺范围限制 |
| `/buyer/member/evaluation*` | 评价详情、创建 | 若当前会员是代理商，仅可访问已审核通过绑定店铺的数据 |

## 5. 联调约束

- 管理端前端接入前，先执行：
  - `DB/patch_wholesale_stage1.sql`
  - `DB/patch_wholesale_stage1_menu.sql`
- 本轮菜单初始化只落已存在后端接口对应的权限项，不引入空菜单权限。
- 代理商不新增独立登录态，继续复用 `MEMBER` 登录体系。
- 数据隔离依据：
  - `agent_role_relation`
  - `agent_store_bind`
  - `audit_status = APPROVED`
  - `bind_status = BOUND`

## 6. 当前已知边界

- 本轮未展开 `admin-web` 页面实现，只冻结后端接口与菜单权限。
- 本轮未进入商品、订单履约、资金对账、统计大盘的完整批发闭环。
- 若后续新增管理端页面或按钮权限，应继续沿用现有菜单 `permission` 前缀，不另起平行命名。
