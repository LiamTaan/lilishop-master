# 批发商城阶段一样例请求清单

## 1. 说明

本清单用于里程碑 `0+1` 的后端联调与冒烟验证，默认调用对象为：

- 管理端：`manager-api`
- 买家端：`buyer-api`

调用前提：

- 已执行 `DB/patch_wholesale_stage1.sql`
- 已执行 `DB/patch_wholesale_stage1_menu.sql`
- 已准备：
  - 一个普通会员
  - 一个代理商会员
  - 一个待审核或已审核店铺

## 2. 管理端样例

### 2.1 创建代理商身份

```http
POST /manager/agent/role
Content-Type: application/json

{
  "memberId": "agent-member-001",
  "regionId": "110000",
  "remark": "华北一区代理商"
}
```

### 2.2 查询代理商身份列表

```http
GET /manager/agent/role
```

### 2.3 创建代理商店铺绑定

```http
POST /manager/agent/store
Content-Type: application/json

{
  "agentMemberId": "agent-member-001",
  "storeId": "store-001",
  "regionId": "110000",
  "auditRemark": "提交平台审核",
  "remark": "首批合作店铺"
}
```

### 2.4 审核代理商店铺绑定

审核通过：

```http
PUT /manager/agent/store/bind-id-001/audit
Content-Type: application/json

{
  "auditStatus": "APPROVED",
  "auditRemark": "审核通过"
}
```

审核拒绝：

```http
PUT /manager/agent/store/bind-id-001/audit
Content-Type: application/json

{
  "auditStatus": "REJECTED",
  "auditRemark": "资料不完整"
}
```

### 2.5 解绑代理商店铺

```http
PUT /manager/agent/store/bind-id-001/unbind
```

### 2.6 店铺审核

```http
PUT /manager/store/store/audit/store-001
Content-Type: application/json

{
  "auditStatus": "APPROVED",
  "auditRemark": "主体资料校验通过"
}
```

可选状态：

- `APPROVED`
- `REJECTED`
- `FROZEN`

### 2.7 查询店铺审核历史

```http
GET /manager/store/store/audit/log/store-001
```

## 3. 买家端样例

### 3.1 查询当前是否代理商

```http
GET /buyer/agent/current/isAgent
```

### 3.2 查询当前代理商已绑定店铺

```http
GET /buyer/agent/current/stores
```

### 3.3 查询代理商订单列表

```http
GET /buyer/order/order?pageNumber=1&pageSize=10
```

预期：

- 若当前会员是代理商，仅返回已审核通过绑定店铺订单
- 若当前会员不是代理商，按原会员订单逻辑返回

### 3.4 查询代理商售后列表

```http
GET /buyer/order/afterSale/page?pageNumber=1&pageSize=10
```

### 3.5 查询代理商投诉列表

```http
GET /buyer/order/complain?pageNumber=1&pageSize=10
```

### 3.6 查询发票列表

```http
GET /buyer/trade/receipt?pageNumber=1&pageSize=10
```

预期：

- 强制只返回当前会员发票
- 代理商额外受绑定店铺范围限制

### 3.7 提交售后申请

```http
POST /buyer/order/afterSale/save/order-item-sn-001
Content-Type: application/x-www-form-urlencoded

orderItemSn=order-item-sn-001
serviceType=RETURN_MONEY
refundWay=ORIGINAL
reason=商品存在质量问题
problemDesc=包装破损且无法正常使用
num=1
```

### 3.8 创建交易投诉

```http
POST /buyer/order/complain
Content-Type: application/x-www-form-urlencoded

orderSn=order-sn-001
skuId=sku-001
content=商家发错货
images=https://example.com/a.png
```

### 3.9 保存发票

```http
POST /buyer/trade/receipt
Content-Type: application/x-www-form-urlencoded

orderSn=order-sn-001
receiptTitle=企业发票
companyName=北京示例贸易有限公司
taxpayerId=91110000123456789X
receiptType=电子普通发票
receiptPhone=13800000000
receiptEmail=test@example.com
receiptContent=商品明细
receiptPrice=199.00
```

预期：

- 服务端会以当前登录会员覆盖 `memberId`
- 不依赖前端传入 `memberId`

## 4. 推荐联调顺序

1. 管理端创建代理商身份
2. 管理端创建代理商店铺绑定
3. 管理端审核绑定通过
4. 管理端完成店铺审核
5. 买家端验证当前会员代理商身份
6. 买家端验证订单列表隔离
7. 买家端验证售后列表与详情隔离
8. 买家端验证投诉与发票隔离
9. 买家端验证评价与订单动作隔离

## 5. 关键验证结论

- 同一代理商重复绑定同一店铺必须失败
- 未审核通过绑定的代理商不能看到目标店铺订单/售后/投诉/发票/评价
- 已审核通过绑定的代理商只能看到绑定店铺数据
- 普通会员不受代理商隔离逻辑影响
