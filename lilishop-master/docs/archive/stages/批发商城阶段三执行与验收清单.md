# 批发商城阶段三执行与验收清单

## 1. 适用范围

本清单用于阶段三“订单、发货、售后、核销主链路”后端落地后的联调准备与验收。

- 当前范围：
  - 订单列表、详情、状态查询
  - 店铺发货、部分发货
  - 售后审核与履约
  - 核销动作与核销记录

## 2. 执行顺序

### 2.1 数据库脚本

按顺序执行以下 SQL：

1. `DB/patch_wholesale_stage3.sql`

执行要求：

- 使用 UTF-8 文件链路导入。
- 执行前建议备份 `li_verification_record`。
- 执行后确认新增表存在：
  - `li_verification_record`

### 2.2 服务编译

建议使用 Java 21：

```powershell
$env:JAVA_HOME='D:\utils\jdk21'
$env:Path='D:\utils\jdk21\bin;' + $env:Path
mvn -pl framework,manager-api,buyer-api,seller-api -am -DskipTests compile
```

## 3. 数据验收项

- `li_verification_record` 表可正常写入核销记录
- 核销记录包含：
  - 订单编号
  - 店铺编号
  - 会员编号
  - 核销码
  - 核销来源
  - 核销结果
  - 核销时间

## 4. 接口验收项

### 4.1 订单发货

- `POST /store/order/order/{orderSn}/delivery`
- `PUT /store/order/order/{orderSn}/deliver`
- `POST /store/order/order/{orderSn}/partDelivery`

### 4.2 核销主链路

- `GET /store/order/order/getOrderByVerificationCode/{verificationCode}`
- `PUT /store/order/order/take/{orderSn}/{verificationCode}`
- `GET /manager/other/verificationRecord`
- `GET /store/order/verificationRecord`
- `GET /buyer/order/verificationRecord`

### 4.3 售后主链路

- 管理端售后分页、审核、退款、收件地址
- 店铺端售后分页、审核、确认收货、收件地址

## 5. 回归检查项

- 普通订单发货不受核销记录新增影响
- 核销成功后可查到对应记录
- 店铺端只能看到本店核销记录
- 买家端只能看到本人核销记录
- 无新增 JDBC、裸 SQL 业务代码
- 无新增业务 `Map<String, Object>` 返回模型扩散到新链路

## 6. 问题排查建议

- 若核销记录为空，先确认订单是否真的触发 `take()`。
- 若店铺端看不到记录，先确认 `storeId` 与订单店铺一致。
- 若编译失败，确认 `JAVA_HOME` 指向 `D:\utils\jdk21`。
