# 批发商城阶段二范围说明

## 阶段来源

- 阶段二对应完整计划中的“商品、批发、积分、卡券、首页分类运营位”
- 里程碑对应：`里程碑2 = 商品与交易入口完成`

## 本次优先落地范围

- 复用既有商品、批发、积分能力：
  - `modules/goods`
  - `modules/promotion`
  - `modules/page`
- 本次新增并已承接的明确缺口：
  - `li_operation_shortcut_nav`
  - `li_card_coupon_goods`

## 接口承接

- 管理端首页分类配置：
  - `GET /manager/other/shortcutNav`
  - `GET /manager/other/shortcutNav/{id}`
  - `POST /manager/other/shortcutNav`
  - `PUT /manager/other/shortcutNav/{id}`
  - `DELETE /manager/other/shortcutNav/{id}`
- 管理端批发商品治理：
  - `GET /manager/goods/goods/wholesale/list`
- 买家端首页分类展示：
  - `GET /buyer/other/shortcutNav`
- 管理端卡券商品：
  - `POST /manager/promotion/cardCouponGoods/bind`
  - `GET /manager/promotion/cardCouponGoods`
  - `DELETE /manager/promotion/cardCouponGoods/{ids}`
- 买家端卡券商品：
  - `GET /buyer/promotion/cardCouponGoods/coupon/{couponId}`
  - `GET /buyer/promotion/cardCouponGoods/sku/{skuId}`
- 买家端代理商批发查询：
  - `GET /buyer/agent/goods/wholesale`
  - `GET /buyer/agent/goods/wholesale/{goodsId}`

说明：

- 当前接口路径、表名、类名仍沿用 `shortcutNav` 技术命名。
- 业务语义按“首页中间分类宫格”理解，不再称为“金刚区快捷入口”。
- 管理端页面名称统一使用“首页分类配置”。
- 若后续重构接口命名，可再将 `shortcutNav` 技术名收口为“首页分类”语义名。
