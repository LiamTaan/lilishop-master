# 批发商城阶段五执行与验收清单

## 1. 适用范围

本清单用于阶段五第一轮“钱包、提现、结算单、资金联调口径”后端落地后的联调准备与验收。

- 当前范围：
  - 买家端钱包余额、提现设置、提现申请、提现记录、钱包流水
  - 平台端钱包查询、手工加款、提现审核、充值记录、钱包流水
  - 店铺端结算单分页、详情、收支流水、分销流水、核对、下载
  - 代理商资金域入口说明与后续新增接口预留

## 2. 执行顺序

### 2.1 数据基线

本阶段第一轮主要复用现有钱包与结算底座表：

- `li_member_wallet`
- `li_member_withdraw_apply`
- `li_wallet_log`
- `li_recharge`
- `li_bill`
- `li_store_flow`

执行要求：

- 保持前序阶段数据库基线一致。
- 若历史环境未初始化钱包数据，先保证测试会员存在钱包记录后再联调。

### 2.2 服务编译

建议使用 Java 21：

```powershell
$env:JAVA_HOME='D:\utils\jdk21'
$env:Path='D:\utils\jdk21\bin;' + $env:Path
mvn -pl framework,manager-api,buyer-api,seller-api -am -DskipTests compile
```

## 3. 数据验收项

- 会员钱包可正常查询余额与冻结金额。
- 提现设置可正确返回最小提现金额、手续费、提现方式。
- 提现申请提交后能生成提现申请记录。
- 平台审核通过/驳回后，提现记录状态可正确流转。
- 店铺结算单可正确返回出账、核对、完成等状态。
- 店铺结算单流水可按 `PAY / REFUND` 正常查询。

## 4. 接口验收项

### 4.1 买家端钱包与提现

- `GET /buyer/wallet/wallet`
- `GET /buyer/wallet/wallet/withdrawalSettingVO`
- `POST /buyer/wallet/wallet/withdrawal`
- `GET /buyer/member/withdrawApply`
- `GET /buyer/wallet/log`

### 4.2 平台端钱包与审核

- `GET /manager/wallet/wallet`
- `PUT /manager/wallet/wallet/increase`
- `GET /manager/wallet/withdrawApply`
- `POST /manager/wallet/withdrawApply`
- `GET /manager/wallet/recharge`
- `GET /manager/wallet/log`

### 4.3 店铺端结算单

- `GET /store/order/bill/getByPage`
- `GET /store/order/bill/get/{id}`
- `GET /store/order/bill/{id}/getStoreFlow`
- `GET /store/order/bill/{id}/getDistributionFlow`
- `PUT /store/order/bill/check/{id}`
- `GET /store/order/bill/downLoad/{id}`

### 4.4 代理商入口回归

- `GET /buyer/agent/role/check`
- `GET /buyer/agent/store`
- `GET /buyer/agent/goods/wholesale`

## 5. 回归检查项

- 普通买家钱包查询与提现功能不受阶段一到四改动影响。
- 平台端提现审核继续走既有 `MemberWithdrawApplyService` 主链路。
- 店铺端结算单查询不受拼团、核销、卡券增强影响。
- 代理商仍在 `buyer-api` 体系内复用登录态，不单独创建资金账号体系。
- 无新增 JDBC、裸 SQL 业务代码。
- 无新增业务 `Map<String, Object>` 返回模型扩散到资金链路。

## 6. 问题排查建议

- 若钱包余额为空，先确认测试会员是否已初始化 `member_wallet`。
- 若提现失败，先确认提现设置、真实姓名、第三方账号等前置参数是否齐备。
- 若平台审核无效，先确认提现记录状态是否允许审核。
- 若结算单流水为空，先确认账单周期内是否存在付款或退款流水。
- 若编译失败，确认 `JAVA_HOME` 指向 `D:\utils\jdk21`。
