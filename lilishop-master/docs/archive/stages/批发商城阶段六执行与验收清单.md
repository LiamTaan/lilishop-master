# 批发商城阶段六执行与验收清单

## 1. 适用范围

本清单用于阶段六“经营概览、统计、平台治理、Swagger 收口”第一轮后端落地后的联调准备与验收。

- 当前范围：
  - 管理端批发商城平台工作台聚合
  - 店铺端经营概览聚合
  - 复用既有统计、排行、订单概览、分账概览能力

## 2. 执行顺序

### 2.1 数据基线

本阶段主要复用现有统计与结算数据：

- `li_store_flow`
- `li_bill`
- `li_wallet_log`
- 统计索引与统计表

执行要求：

- 保持阶段一到五数据库基线一致。
- 如统计数据为空，先确认环境是否已有订单、流水、店铺、商品基础数据。

### 2.2 服务编译

建议使用 Java 21：

```powershell
$env:JAVA_HOME='D:\utils\jdk21'
$env:Path='D:\utils\jdk21\bin;' + $env:Path
mvn -pl framework,manager-api,seller-api -am -DskipTests compile
```

## 3. 数据验收项

- 平台工作台可返回首页基础概览、热卖店铺排行、热卖商品排行。
- 平台工作台可返回阶段五已有的分账余额概览。
- 平台工作台可返回钱包流水口径下的代理商资金概览。
- 店铺经营概览可返回首页概览、订单概览、热卖商品排行。

## 4. 接口验收项

### 4.1 平台工作台聚合

- `GET /manager/dashboard/wholesale`

### 4.2 店铺经营概览聚合

- `GET /store/dashboard/overview`

### 4.3 复用统计回归

- `GET /manager/statistics/index`
- `GET /manager/statistics/index/goodsStatistics`
- `GET /manager/statistics/index/storeStatistics`
- `GET /store/statistics/index`
- `GET /store/statistics/order/overview`
- `GET /store/statistics/goods`

## 5. 回归检查项

- 平台工作台聚合不应影响既有 `/manager/statistics/*` 能力。
- 店铺经营概览聚合不应影响既有 `/store/statistics/*` 能力。
- 分账余额概览与阶段五平台分账接口口径保持一致。
- 无新增 JDBC、裸 SQL 业务代码。
- 无新增业务 `Map<String, Object>` 返回模型扩散到新聚合接口。

## 6. 问题排查建议

- 若平台工作台排行为空，先确认热卖商品与店铺统计在原统计接口中是否正常。
- 若代理商资金概览为 0，先确认钱包流水表是否已有测试数据。
- 若店铺经营概览为空，先确认当前登录店铺是否已有订单与商品统计数据。
- 若编译失败，确认 `JAVA_HOME` 指向 `D:\utils\jdk21`。
