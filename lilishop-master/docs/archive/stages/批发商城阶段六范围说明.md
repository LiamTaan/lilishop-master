# 批发商城阶段六范围说明

## 阶段来源

- 阶段六对应完整计划中的“经营概览、统计、平台治理、Swagger 收口”
- 里程碑对应：`里程碑6 = 管理驾驶舱与治理收口`

## 当前复用底座

- `modules/statistics`
- `modules/store`
- `modules/order/order`
- `modules/agent`
- `modules/wallet`

## 当前已存在可直接承接能力

- 管理端统计：
  - `GET /manager/statistics/index`
  - `GET /manager/statistics/index/goodsStatistics`
  - `GET /manager/statistics/index/storeStatistics`
  - `GET /manager/statistics/order/overview`
  - `GET /manager/statistics/overview`
- 店铺端统计：
  - `GET /store/statistics/index`
  - `GET /store/statistics/order/overview`
  - `GET /store/statistics/goods`
  - `GET /store/statistics/refundOrder`
- 阶段五新增治理：
  - `GET /manager/profitsharing/record`
  - `GET /manager/profitsharing/balance`
  - `GET /manager/profitsharing/rule`
  - `POST /manager/profitsharing/rule`
  - `PUT /manager/profitsharing/rule/{id}`
  - `DELETE /manager/profitsharing/rule/{id}`

## 本轮新增承接

- 批发商城平台工作台聚合接口：
  - `GET /manager/dashboard/wholesale`
- 批发商城店铺经营概览聚合接口：
  - `GET /store/dashboard/overview`
- 聚合内容：
  - 首页基础概览
  - 热卖店铺排行
  - 热卖商品排行
  - 类目销售排行
  - 平台分账余额概览
  - 代理商资金概览
  - 代理商经营概览

## 本轮落地策略

- 不重造现有统计域，优先复用 `IndexStatisticsService`
- 不新建平台驾驶舱中间表，先以现有统计、账单、钱包流水聚合为主
- 统一聚合接口已交付，分页排行、筛选条件、时间维度作为二期增强项

## 管理端页面命名口径

- `平台工作台`：优先对应 `GET /manager/dashboard/wholesale`
- `平台经营概览`：优先对应既有 `/manager/statistics/index`
- `门店排行`：优先对应 `/manager/statistics/index/storeStatistics`
- `商品/品类排行`：优先对应 `/manager/statistics/index/goodsStatistics`
- `订单概览`：优先对应 `/manager/statistics/order/overview`

说明：

- “平台工作台”与“平台经营概览”不是同一页面名。
- `平台工作台` 强调批发商城治理聚合首页。
- `平台经营概览` 强调既有统计域指标页。
- 后续管理端前端接入时，不再混用“数据概览 / 工作台 / 经营概览”三个词指向同一个页面。

- 二期可继续扩展平台端“类目排行 / 拼团成团率 / 代理商概览”治理聚合接口
- 阶段六执行清单、样例请求、联调 HTTP 文档已补齐
