# 批发商城阶段四执行与验收清单

## 1. 适用范围

本清单用于阶段四“拼团、营销、优惠券”后端落地后的联调准备与验收。

- 当前范围：
  - 买家端拼团商品列表、详情、待成团成员、分享信息
  - 店铺端拼团活动新增、编辑、上下线、删除、活动商品查询
  - 平台端拼团活动分页、详情、商品列表、成员查询、状态操作
  - 优惠券与满减链路复用现有 `promotion` 模块接口

## 2. 执行顺序

### 2.1 数据基线

本阶段主要复用现有 `li_pintuan`、`li_promotion_goods`、`li_coupon`、`li_full_discount` 等营销表。

执行要求：

- 延续阶段一至三脚本后的数据库基线。
- 如环境缺少拼团菜单或历史数据异常，先补齐既有初始化数据后再联调。

### 2.2 服务编译

建议使用 Java 21：

```powershell
$env:JAVA_HOME='D:\utils\jdk21'
$env:Path='D:\utils\jdk21\bin;' + $env:Path
mvn -pl framework,manager-api,buyer-api,seller-api -am -DskipTests compile
```

## 3. 数据验收项

- 拼团活动可正常读取活动基础信息、状态、时间范围、店铺归属。
- 拼团活动商品列表可按 `promotionId + promotionType=PINTUAN` 正常返回。
- 待成团成员列表可返回当前拼团活动下的未成团成员。
- 店铺端新增/编辑拼团时，`scopeId` 能随 SKU 列表同步更新。
- 优惠券与满减活动不受本轮拼团增强影响。

## 4. 接口验收项

### 4.1 买家端拼团

- `GET /buyer/promotion/pintuan`
- `GET /buyer/promotion/pintuan/{pintuanId}`
- `GET /buyer/promotion/pintuan/{pintuanId}/members`
- `GET /buyer/promotion/pintuan/share`

### 4.2 店铺端拼团

- `GET /store/promotion/pintuan`
- `GET /store/promotion/pintuan/{id}`
- `GET /store/promotion/pintuan/goods/{pintuanId}`
- `GET /store/promotion/pintuan/{pintuanId}/members`
- `POST /store/promotion/pintuan`
- `PUT /store/promotion/pintuan`
- `PUT /store/promotion/pintuan/status/{pintuanId}`
- `DELETE /store/promotion/pintuan/{pintuanId}`

### 4.3 平台端拼团

- `GET /manager/promotion/pintuan`
- `GET /manager/promotion/pintuan/{id}`
- `GET /manager/promotion/pintuan/goods/{pintuanId}`
- `GET /manager/promotion/pintuan/{pintuanId}/members`
- `PUT /manager/promotion/pintuan/status/{pintuanIds}`

### 4.4 营销复用回归

- 管理端优惠券：`/manager/promotion/coupon*`
- 买家端优惠券：`/buyer/promotion/coupon*`
- 管理端满减：`/manager/promotion/fullDiscount*`
- 店铺端满减：`/store/promotion/fullDiscount*`

## 5. 回归检查项

- 买家端仅能查询进行中拼团商品，不应误出未开始或已结束活动。
- 店铺端只能维护本店拼团活动。
- 平台端可以查看全局拼团活动并调整状态。
- 拼团活动详情、成员查询、分享查询接口返回结构保持兼容。
- 无新增 JDBC、裸 SQL 业务代码。
- 无新增业务 `Map<String, Object>` 返回模型扩散到本阶段新增链路。

## 6. 问题排查建议

- 若买家端拼团列表为空，先确认 `promotionStatus=START` 的活动是否存在。
- 若店铺端编辑后商品范围异常，先确认 `promotionGoodsList` 是否携带 `skuId`。
- 若成员列表为空，先确认是否存在未成团订单以及对应的团长订单。
- 若编译失败，确认 `JAVA_HOME` 指向 `D:\utils\jdk21`。
