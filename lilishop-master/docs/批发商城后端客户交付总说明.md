本文档用于对外说明当前批发商城后端正式交付状态，适合用于：

- 客户阶段性汇报

## 4. 当前已明确的核心状态流转

### 4.1 店铺入驻与审核

- `DRAFT -> SUBMITTED -> APPROVED`
- `SUBMITTED -> REJECTED`
- `APPROVED -> FROZEN`
- `FROZEN -> APPROVED`

### 4.2 代理商绑定

- 创建申请：进入 `SUBMITTED`
- 审核通过：`SUBMITTED -> APPROVED`，绑定状态进入 `BOUND`
- 审核驳回：`SUBMITTED -> REJECTED`
- 后台解绑：绑定状态进入 `UNBOUND`

### 4.3 订单

- 下单：`UNPAID`
- 支付成功：`UNPAID -> PAID`
- 普通物流履约：`PAID -> UNDELIVERED -> DELIVERED -> COMPLETED`
- 自提履约：`PAID -> STAY_PICKED_UP -> COMPLETED`
- 核销履约：`PAID -> TAKE -> COMPLETED`
- 取消：进入 `CANCELLED`

### 4.4 售后

- 提交售后：进入 `APPLY`
- 审核通过：
  - 原路退款：直接 `COMPLETE`
  - 线下退款：进入 `WAIT_REFUND`
  - 退货退款：进入 `PASS`
- 审核驳回：`REFUSE`
- 买家寄回：`PASS -> BUYER_RETURN`
- 卖家确认收货后：
  - 原路退款：`COMPLETE`
  - 线下退款：`WAIT_REFUND`
  - 拒收终止：`SELLER_TERMINATION`
- 平台退款：`WAIT_REFUND -> COMPLETE`

### 4.5 分账与结算

- 账单出账：`OUT`
- 商家核对：`OUT -> CHECK`
- 平台完成：`CHECK -> COMPLETE`

### 4.6 核销与异常核销

- 核销结果：`SUCCESS / FAIL`
- 异常核销当前口径：
  - `resultType = FAIL`
  - 且 `remark` 非空

## 5. 当前后端交付边界

当前后端已经具备“治理层正式交付”的主体能力，但仍需明确边界：

- 当前重点已放在后端治理接口、状态流转、汇总口径、导出查询口径
- 当前不是重新开发一套基础商城底座，而是在现有 Lilishop 基础上完成批发商城业务治理收口
- 当前管理端是否全部完成页面联调，取决于前端实际接入进度，而不是后端接口是否存在

也就是说，后端现在已经比较适合进入：

- 管理端联调
- 客户阶段验收
- 测试按治理域逐项验证

## 6. 当前建议的客户验收顺序

建议客户按以下顺序验收：

1. 平台工作台是否能直接展示经营与治理总览
2. 店铺、代理商、分账、对账、订单、售后、核销是否都有分页与汇总
3. 店铺审核、代理绑定、售后是否具备明确状态流转
4. 分账、采购对账、资金对账是否具备总览、汇总、详情、导出查询
5. 异常核销是否能单独分页、汇总、导出查询

## 7. 当前建议的下一步

建议从下面两条路线里选一条继续推进：

### 路线 A：进入前后端联调

- 以前端真实页面为准补字段
