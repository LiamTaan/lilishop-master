# 批发商城后端状态流转与治理动作说明

## 1. 文档目的

本文档用于补齐批发商城后端正式交付中的“状态流转”和“治理动作”口径，解决以下问题：

- 客户验收时确认每个业务域是否具备完整状态闭环
- 前后端联调时统一状态值含义
- 测试执行时明确前置条件、动作结果和异常场景

本文档只描述当前仓库中已经落地或已经明确采用的后端状态口径。

## 2. 店铺入驻与治理

状态枚举：

- `DRAFT`：草稿
- `SUBMITTED`：待审核
- `APPROVED`：审核通过
- `REJECTED`：审核驳回
- `FROZEN`：已冻结

对应代码：

- [StoreAuditStatusEnum](/D:/user_wuyou/lilishop-master/lilishop-master/framework/src/main/java/cn/lili/modules/store/entity/enums/StoreAuditStatusEnum.java)

治理动作：

- 店铺提交申请：`DRAFT/REJECTED -> SUBMITTED`
- 平台审核通过：`SUBMITTED -> APPROVED`
- 平台审核驳回：`SUBMITTED -> REJECTED`
- 平台冻结：`APPROVED -> FROZEN`
- 平台启用：`FROZEN -> APPROVED`

相关接口：

- `PUT /manager/store/store/audit/{id}`
- `GET /manager/store/store/audit/log/{storeId}`
- `PUT /manager/store/store/disable/{id}`
- `PUT /manager/store/store/enable/{id}`

治理要求：

- 每次审核动作必须记录审核备注
- 审核历史必须可追溯
- 冻结后不得继续按营业中店铺处理

## 3. 代理商店铺绑定治理

审核状态：

- `SUBMITTED`：待审核
- `APPROVED`：审核通过
- `REJECTED`：审核驳回

绑定状态：

- `BOUND`：已绑定
- `UNBOUND`：已解绑
- `DISABLED`：已停用

对应代码：

- [AgentBindAuditStatusEnum](/D:/user_wuyou/lilishop-master/lilishop-master/framework/src/main/java/cn/lili/modules/agent/entity/enums/AgentBindAuditStatusEnum.java)
- [AgentStoreBindStatusEnum](/D:/user_wuyou/lilishop-master/lilishop-master/framework/src/main/java/cn/lili/modules/agent/entity/enums/AgentStoreBindStatusEnum.java)

治理动作：

- 创建绑定申请：进入 `SUBMITTED`
- 平台审核通过：`SUBMITTED -> APPROVED`，绑定态进入 `BOUND`
- 平台审核驳回：`SUBMITTED -> REJECTED`
- 平台解绑：绑定态进入 `UNBOUND`

相关接口：

- `POST /manager/agent/store`
- `GET /manager/agent/store`
- `PUT /manager/agent/store/{bindId}/audit`
- `PUT /manager/agent/store/{bindId}/unbind`

## 4. 订单治理

状态枚举：

- `UNPAID`：未付款
- `PAID`：已付款
- `UNDELIVERED`：待发货
- `PARTS_DELIVERED`：部分发货
- `DELIVERED`：已发货
- `STAY_PICKED_UP`：待自提
- `TAKE`：待核验
- `COMPLETED`：已完成
- `CANCELLED`：已关闭

对应代码：

- [OrderStatusEnum](/D:/user_wuyou/lilishop-master/lilishop-master/framework/src/main/java/cn/lili/modules/order/order/entity/enums/OrderStatusEnum.java)

主链路动作：

- 下单创建：进入 `UNPAID`
- 支付成功：`UNPAID -> PAID`
- 普通物流单确认：`PAID -> UNDELIVERED`
- 自提单确认：`PAID -> STAY_PICKED_UP`
- 虚拟/核销单确认：`PAID -> TAKE`
- 发货成功：`UNDELIVERED -> DELIVERED`
- 核销成功：`TAKE/STAY_PICKED_UP -> COMPLETED`
- 订单取消：`UNPAID/PAID/部分可取消状态 -> CANCELLED`
- 完成订单：`DELIVERED -> COMPLETED`

治理动作：

- 平台确认收款
- 平台修改收货信息
- 平台修改价格
- 平台取消订单
- 平台查看物流轨迹
- 平台写卖家备注

相关接口：

- `GET /manager/order/order`
- `GET /manager/order/order/summary`
- `GET /manager/order/order/{orderSn}`
- `POST /manager/order/order/{orderSn}/pay`
- `POST /manager/order/order/{orderSn}/cancel`

## 5. 售后治理

状态枚举：

- `APPLY`：申请中
- `PASS`：已通过
- `REFUSE`：已拒绝
- `BUYER_RETURN`：待卖家收货
- `SELLER_CONFIRM`：卖家确认收货
- `SELLER_TERMINATION`：卖家终止售后
- `BUYER_CANCEL`：买家取消售后
- `WAIT_REFUND`：等待平台退款
- `COMPLETE`：已完成

对应代码：

- [AfterSaleStatusEnum](/D:/user_wuyou/lilishop-master/lilishop-master/framework/src/main/java/cn/lili/modules/order/trade/entity/enums/AfterSaleStatusEnum.java)

主链路动作：

- 买家提交售后：进入 `APPLY`
- 平台审核通过：
  - 仅退款原路退回：直接进入 `COMPLETE`
  - 仅退款线下退回：进入 `WAIT_REFUND`
  - 退货退款：进入 `PASS`
- 平台审核拒绝：`APPLY -> REFUSE`
- 买家寄回：`PASS -> BUYER_RETURN`
- 商家确认收货：
  - 原路退款：`BUYER_RETURN -> COMPLETE`
  - 线下退款：`BUYER_RETURN -> WAIT_REFUND`
  - 拒收终止：`BUYER_RETURN -> SELLER_TERMINATION`
- 平台退款：`WAIT_REFUND -> COMPLETE`
- 买家取消：`APPLY/PASS -> BUYER_CANCEL`

治理动作：

- 售后分页
- 售后汇总
- 售后审核
- 售后退款
- 售后物流追踪

相关接口：

- `GET /manager/order/afterSale/page`
- `GET /manager/order/afterSale/summary`
- `PUT /manager/order/afterSale/review/{afterSaleSn}`
- `PUT /manager/order/afterSale/refund/{afterSaleSn}`

## 6. 分账与结算治理

账单状态：

- `OUT`：已出账
- `CHECK`：已核对
- `COMPLETE`：已完成

对应代码：

- [BillStatusEnum](/D:/user_wuyou/lilishop-master/lilishop-master/framework/src/main/java/cn/lili/modules/store/entity/enums/BillStatusEnum.java)

分账处理状态：

- `ORDER_CANCEL`：订单取消
- `WAIT_COMPLETE`：待订单完成
- `PROCESSING`：处理中
- `FINISHED`：分账完成

对应代码：

- [ProfitSharingStatusEnum](/D:/user_wuyou/lilishop-master/lilishop-master/framework/src/main/java/cn/lili/modules/order/order/entity/enums/ProfitSharingStatusEnum.java)

治理动作：

- 账单出账：进入 `OUT`
- 商家核对：`OUT -> CHECK`
- 平台完成结算：`CHECK -> COMPLETE`
- 分账记录按账单状态映射为处理中或完成

相关接口：

- `GET /manager/profitsharing/record`
- `GET /manager/profitsharing/summary`
- `GET /manager/profitsharing/balance`

## 8. 异常核销治理

当前异常识别口径：

- `resultType = FAIL`
- 且 `remark` 非空

当前异常治理汇总维度：

- 异常核销总数
- 店铺端异常数
- 平台端异常数
- 买家端异常数

治理动作：

- 平台分页查看异常核销
- 平台导出异常核销清单
- 平台查看异常核销汇总

相关接口：

- `GET /manager/other/verificationRecord/exception`
- `GET /manager/other/verificationRecord/exception/list`
- `GET /manager/other/verificationRecord/exception/summary`

## 10. 验收建议

建议客户验收时按以下顺序核对：

1. 平台工作台是否能直接展示治理总览
2. 店铺、代理商、分账、对账、订单、售后、核销是否都有分页+汇总
3. 售后、店铺审核、代理绑定是否都有明确状态流转
4. 异常核销是否能单独拉取分页、汇总和导出查询
5. 所有关键治理动作是否有备注、历史或异常说明能力
