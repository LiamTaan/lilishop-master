# 批发商城管理端接口总对照表

## 1. 文档目的

本文档用于把管理端主页面、主接口、汇总接口、关键动作接口统一到一张表里，方便前端接入、测试联调和客户核对。

## 2. 接口总对照

| 页面 | 列表/主接口 | 汇总接口 | 详情接口 | 关键动作接口 |
|---|---|---|---|---|
| 平台工作台 | `GET /manager/dashboard/wholesale` | 同主接口聚合返回 | 无独立详情 | 无独立动作 |
| 店铺入驻审核 | `GET /manager/store/store` | `GET /manager/store/store/summary` | `GET /manager/store/store/get/detail/{storeId}` | `PUT /manager/store/store/audit/{id}` |
| 店铺管理 | `GET /manager/store/store` | `GET /manager/store/store/summary` | `GET /manager/store/store/get/detail/{storeId}` | `PUT /manager/store/store/disable/{id}`、`PUT /manager/store/store/enable/{id}` |
| 店铺审核历史 | 无独立列表 | 无独立汇总 | `GET /manager/store/store/audit/log/{storeId}` | 无独立动作 |
| 代理商治理 | `GET /manager/agent/role`、`GET /manager/agent/store` | 无独立汇总 | 绑定明细以列表字段为主 | `POST /manager/agent/role`、`POST /manager/agent/store`、`PUT /manager/agent/store/{bindId}/audit`、`PUT /manager/agent/store/{bindId}/unbind` |
| 订单管理 | `GET /manager/order/order` | `GET /manager/order/order/summary` | `GET /manager/order/order/{orderSn}` | `POST /manager/order/order/{orderSn}/pay`、`POST /manager/order/order/{orderSn}/cancel`、`PUT /manager/order/order/{orderSn}/sellerRemark` |
| 售后管理 | `GET /manager/order/afterSale/page` | `GET /manager/order/afterSale/summary` | `GET /manager/order/afterSale/get/{sn}` | `PUT /manager/order/afterSale/review/{afterSaleSn}`、`PUT /manager/order/afterSale/refund/{afterSaleSn}` |
| 核销记录 | `GET /manager/other/verificationRecord` | `GET /manager/other/verificationRecord/summary` | 以列表明细字段展示为主 | 无独立动作 |
| 异常核销 | `GET /manager/other/verificationRecord/exception` | `GET /manager/other/verificationRecord/exception/summary` | 以列表明细字段展示为主 | 无独立动作 |
| 分账规则 | `GET /manager/profitsharing/rule` | 无独立汇总 | 以列表字段展示为主 | 规则维护沿现有分账规则能力接入 |
| 分账明细 | `GET /manager/profitsharing/record` | `GET /manager/profitsharing/summary`、`GET /manager/profitsharing/balance` | 以记录字段展示为主 | 导出查询：`GET /manager/profitsharing/record/list` |
| 采购对账 | `GET /manager/agent/reconciliation/{agentMemberId}/procurement` | `GET /manager/agent/reconciliation/overview`、`GET /manager/agent/reconciliation/{agentMemberId}/procurement/summary` | `GET /manager/agent/reconciliation/{agentMemberId}/procurement/{id}` | 导出查询：`GET /manager/agent/reconciliation/{agentMemberId}/procurement/list` |
| 资金对账 | `GET /manager/agent/reconciliation/{agentMemberId}/fund` | `GET /manager/agent/reconciliation/overview`、`GET /manager/agent/reconciliation/{agentMemberId}/fund/summary` | `GET /manager/agent/reconciliation/{agentMemberId}/fund/{id}` | 导出查询：`GET /manager/agent/reconciliation/{agentMemberId}/fund/list` |

## 3. 页面接入说明

- 店铺页、订单页、售后页、核销页优先先接列表和汇总，再补详情和动作。
- 分账、采购对账、资金对账优先先接汇总和分页，再补详情与导出。
- 平台工作台属于单接口聚合页，前端不应拆成多接口拼装。

## 4. 联调要求

- 所有状态型页面都必须展示状态字段、备注字段、时间字段。
- 所有详情页都必须保留主键编号、状态、金额、时间、操作人相关字段。
- 所有动作页都必须在操作前刷新详情或当前行状态，避免前端使用过期状态提交。
