# 首页与工作台接口对接说明

本文整理消费者/代理首页、供货商工作台的接口对接信息，面向 App/H5 首屏联调。

服务划分：

| 端 | 服务 | 本地地址 | 说明 |
| --- | --- | --- | --- |
| 消费者首页 | `buyer-api` | `http://127.0.0.1:8888` | 普通买家首页、搜索、商品流、活动 |
| 代理首页 | `buyer-api` | `http://127.0.0.1:8888` | 代理身份仍走买家端，额外使用代理商接口 |
| 供货商工作台 | `seller-api` | `http://127.0.0.1:8889` | 店铺/供货商登录后的工作台、商品、订单、资产 |

## 一、统一约定

### 1. 统一响应结构

所有接口默认返回 `ResultMessage<T>`：

```json
{
  "success": true,
  "message": "success",
  "code": 200,
  "timestamp": 1710000000000,
  "result": {}
}
```

前端业务数据统一读取 `result`。接口失败时优先根据 `code` 判断失败原因，`message` 只作为提示文案。

### 2. 客户端类型

首页接口常用参数 `clientType`：

| 值 | 说明 |
| --- | --- |
| `H5` | 移动 H5 |
| `APP` | App |
| `WECHAT_MP` | 小程序 |
| `PC` | PC |

结构化首页接口会自动把空值或非法值降级为 `H5`。当 `clientType=APP` 且没有 App 装修数据时，会回退读取 `H5` 首页装修数据。

### 3. 登录要求

当前 `buyer-api/src/main/resources/application.yml` 白名单已放开：

- `/buyer/other/pageData/**`
- `/buyer/other/shortcutNav/**`
- `/buyer/goods/**`
- `/buyer/store/**`
- `/buyer/promotion/pintuan/**`
- `/buyer/promotion/seckill/**`

结构化首页主接口 `/buyer/other/home/platform` 当前未在白名单中。实际接法有两种：

- 已登录首页：请求头携带登录 `accessToken`，接口会返回带用户行为推荐的数据。
- 未登录也要展示首页：建议后端将 `/buyer/other/home/**` 加入白名单；未登录时个性化模块会按热门/新品等兜底规则返回。

## 二、消费者/代理首页主接口

### 获取结构化平台首页

```http
GET /buyer/other/home/platform?clientType=H5
```

Controller：`PlatformHomeBuyerController`

用途：一次性返回截图首页中大部分模块，推荐作为首页首屏主接口。

请求参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `clientType` | string | 是 | `H5`、`APP`、`WECHAT_MP`、`PC` |

返回 `result` 字段：

| 字段 | 类型 | 页面位置 | 说明 |
| --- | --- | --- | --- |
| `clientType` | string | - | 实际生效的客户端类型 |
| `banners` | array | 顶部轮播/广告图 | 来自首页装修的 `carousel`、`topAdvert` |
| `shortcutNavList` | array | 十宫格分类入口 | 来自首页快捷入口配置 |
| `floorModules` | array | 首页运营楼层 | 来自首页装修 `wholesaleFloor` |
| `seckill` | object/null | 限时秒杀 | 当前/最近秒杀时段与商品 |
| `monthlyHotGoods` | array | 本月热销 | 按本月商品统计销量取商品 |
| `newGoods` | array | 时令上新/新品 | 按商品创建时间倒序取商品 |
| `frequentStores` | array | 常买店铺 | 登录用户按订单/收藏/足迹推荐，未登录按热店兜底 |
| `guessYouLikeGoods` | array | 猜你喜欢 | 登录用户按行为推荐，冷启动用热门/新品 |
| `lowPriceZoneGoods` | array | 低价专区 | 价格上限默认 99 元，可被装修配置覆盖 |
| `todaySpecialGoods` | array | 今日特惠 | 默认取秒杀、拼团、砍价等活动商品 |
| `recommendationModules` | array | 推荐 Tab 聚合 | 已按配置排序聚合常买店铺/猜你喜欢/低价/特惠 |

## 四、补充接口

### 1. 装修首页原始数据

```http
GET /buyer/other/pageData/getIndex?clientType=H5
```

返回：

```json
{
  "pageData": "{\"list\":[...]}"
}
```

说明：

- 这是旧版装修页接口，`result.pageData` 是字符串 JSON。
- 适合前端已有装修渲染器时直接解析。
- 新 App 页面如果按固定 UI 开发，优先使用结构化首页主接口。

### 2. 首页快捷入口独立接口

```http
GET /buyer/other/shortcutNav?clientType=H5
```

返回 `OperationShortcutNav[]`，字段比结构化首页多：

| 字段 | 说明 |
| --- | --- |
| `id` | 入口 ID |
| `title` / `subtitle` | 标题/副标题 |
| `image` | 图标 |
| `linkType` / `linkValue` | 跳转类型和值 |
| `clientType` | 客户端类型 |
| `sortOrder` | 排序 |
| `displayStatus` | `OPEN` 展示 |
| `remark` | 备注 |

### 3. 商品分类

```http
GET /buyer/goods/category/get/0
```

参数：

| 参数 | 说明 |
| --- | --- |
| `parentId` | 路径参数。`0` 表示全部顶级分类，其他值表示查询某个分类的子级 |

用途：分类页、搜索筛选页、快捷入口跳分类页时使用。

### 4. 搜索热词

```http
GET /buyer/goods/goods/hot-words?count=10
```

返回：`List<String>`。

用途：搜索框占位词、搜索页热门词。

### 5. 商品搜索/瀑布流

```http
GET /buyer/goods/goods/es?pageNumber=1&pageSize=10&keyword=苹果&categoryId=xxx&sort=price&order=asc
```

常用参数：

| 参数 | 说明 |
| --- | --- |
| `pageNumber` | 页码，从 1 开始 |
| `pageSize` | 每页条数 |
| `keyword` | 搜索关键词 |
| `categoryId` | 分类 ID |
| `brandId` | 品牌 ID，多个用 `@` 拼接 |
| `price` | 价格区间，如 `10_30` |
| `storeId` | 店铺 ID，查店铺内商品时使用 |
| `promotionType` | 促销类型 |
| `salesModel` | 销售模式，如批发商品可传 `WHOLESALE` |
| `sort` / `order` | 排序字段和方向 |

用途：首页商品流分页、搜索页、分类页。返回 ES 商品索引分页，前端读取 `result.records`、`result.total` 等分页字段。

### 6. 商品详情

```http
GET /buyer/goods/goods/get/{goodsId}
GET /buyer/goods/goods/sku/{goodsId}/{skuId}
```

用途：

- 点击首页普通商品卡片：用 `goodsId` 跳商品详情。
- 点击有明确促销 SKU 的卡片：可带 `skuId` 获取 SKU 详情。

### 7. 秒杀独立接口

```http
GET /buyer/promotion/seckill
GET /buyer/promotion/seckill/{timeline}
```

说明：

- `/buyer/promotion/seckill` 返回当天秒杀时段和每个时段商品。
- `/buyer/promotion/seckill/{timeline}` 返回指定时段商品。
- 如果首页只展示一个秒杀模块，主接口的 `result.seckill` 已经够用；进入秒杀更多页再调独立接口。

### 8. 拼团商品

```http
GET /buyer/promotion/pintuan?pageNumber=1&pageSize=10&goodsName=蛋糕&categoryPath=xxx
```

参数：

| 参数 | 说明 |
| --- | --- |
| `goodsName` | 商品名称模糊搜索 |
| `categoryPath` | 商品分类路径 |
| `pageNumber` / `pageSize` | 分页 |

后端固定筛选：

- `promotionType=PINTUAN`
- `promotionStatus=START`

用途：截图中的“精品好团”和拼团更多页。

### 9. 店铺列表/店铺详情

```http
GET /buyer/store/store?pageNumber=1&pageSize=10&storeName=乐事
GET /buyer/store/store/get/detail/{storeId}
GET /buyer/store/store/label/get/{storeId}
```

用途：

- 常买店铺模块点击店铺卡片：跳店铺详情。
- 查看更多店铺：调店铺分页。
- 店铺内分类：调店铺商品分类。

### 10. 加入购物车

```http
POST /buyer/trade/carts
Content-Type: application/json

{
  "skuId": "xxx",
  "num": 1,
  "cartType": "CART"
}
```

说明：

- 首页商品卡片的加号按钮需要用户已登录。
- 普通加入购物车 `cartType=CART`。
- 拼团、积分等特殊购买路径需要按对应详情页/结算流程传值。

### 11. 代理商批发商品

如果当前首页是代理商身份首页，批发商品列表建议使用代理商范围接口：

```http
GET /buyer/agent/goods/wholesale?pageNumber=1&pageSize=10&goodsName=苹果
GET /buyer/agent/goods/wholesale/{goodsId}
```

说明：

- 需要登录且当前用户必须是代理商。
- 后端会按代理商可见店铺范围过滤，并固定 `salesModel=WHOLESALE`、审核通过、已上架。

## 五、跳转规则建议

首页中 `banner`、`shortcutNav`、`floorModules` 都使用 `linkType + linkValue` 表达跳转。

建议前端统一封装跳转：

| `linkType` | 建议动作 |
| --- | --- |
| `CATEGORY` | 跳分类/搜索结果页，`linkValue` 为分类 ID |
| `GOODS_DETAIL` | 跳商品详情，`linkValue` 为商品 ID |
| `STORE` | 跳店铺首页，`linkValue` 为店铺 ID |
| `COUPON` | 跳优惠券领取页或直接请求领券 |
| `SPECIAL` | 跳专题页，`linkValue` 为专题 ID |
| `ACTIVITY` | 跳活动页，如秒杀/拼团 |
| `CUSTOM_URL` | 打开外链或内部路由 |

后端没有强制枚举所有历史值，前端要对未知 `linkType` 做兜底：不跳转或打开 `linkValue` 指向的 URL。

## 六、推荐联调顺序

1. 首页首屏：`GET /buyer/other/home/platform?clientType=H5`。
2. 搜索框热词：`GET /buyer/goods/goods/hot-words?count=10`。
3. 商品瀑布流分页：`GET /buyer/goods/goods/es?pageNumber=1&pageSize=10`。
4. 精品好团更多：`GET /buyer/promotion/pintuan?pageNumber=1&pageSize=10`。
5. 商品卡片点击：`GET /buyer/goods/goods/get/{goodsId}`。
6. 店铺卡片点击：`GET /buyer/store/store/get/detail/{storeId}`。
7. 加购按钮：登录后 `POST /buyer/trade/carts`。

## 七、注意事项

- 首页主接口当前是聚合接口，首屏建议只调一次，避免 Banner、快捷入口、推荐商品分散多次请求。
- 如果产品要求匿名用户打开 App 就能看到首页，需要确认 `/buyer/other/home/**` 是否已放入网关/服务白名单。
- `recommendationModules` 已按后端配置排序，做 Tab 时优先使用它；如果页面固定三个 Tab，也可以直接取 `guessYouLikeGoods`、`lowPriceZoneGoods`、`todaySpecialGoods`。
- 秒杀倒计时使用服务端返回的 `startTime` 和 `distanceStartTime`，不要只依赖客户端时间。
- 商品价格字段为 `Double`，前端展示金额时统一格式化到 2 位小数。
- `pageData/getIndex` 返回的是字符串 JSON，只有继续使用装修渲染器时才建议直接解析；新页面尽量使用结构化首页接口，字段更稳定。

## 八、供货商工作台接口

供货商工作台对应截图中的“供货-工作台”，接口服务为 `seller-api`，默认地址 `http://127.0.0.1:8889`。除登录和刷新 token 外，店铺端接口都需要供货商/店铺登录态。

### 1. 登录要求

供货商端免登录白名单主要是：

- `/store/passport/login/**`
- `/store/passport/login/refresh/**`
- `/v3/api-docs`

工作台接口均需要请求头携带店铺端 token。后端通过 `UserContext.getCurrentUser().getStoreId()` 获取当前供货商店铺 ID，不需要前端额外传 `storeId`。

### 2. 工作台主接口

```http
GET /store/dashboard/overview
```

Controller：`StoreDashboardController`

用途：返回工作台经营概览，覆盖截图上方统计卡片和热卖排行数据。

返回 `result` 字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `storeIndexStatistics` | object | 店铺首页概览，包括商品总数、订单总数、订单总额、待处理事项 |
| `orderOverview` | object | 订单经营概览，包括下单、付款、退款、转化率 |
| `goodsRankList` | array | 热卖商品排行，默认按销售金额取前 10 |

`storeIndexStatistics` 字段：

| 字段 | 页面含义 | 说明 |
| --- | --- | --- |
| `goodsNum` | 在线商品/商品总数 | 商品总数量 |
| `orderNum` | 订单总量 | 订单总数量 |
| `orderPrice` | 营业额 | 订单总额 |
| `storeUV` | 访客数 | 店铺 UV |
| `unPaidOrder` | 待付款 | 待付款订单数量 |
| `unDeliveredOrder` | 待发货 | 待发货订单数量 |
| `deliveredOrder` | 待收货 | 待收货订单数量 |
| `returnGoods` | 待处理退货 | 待处理退货数量 |
| `returnMoney` | 待处理退款 | 待处理退款数量 |
| `memberEvaluation` | 待回复评价 | 待回复评价数量 |
| `complaint` | 交易投诉 | 待处理交易投诉数量 |
| `waitUpper` | 待上架商品 | 待上架商品数量 |
| `waitAuth` | 待审核商品 | 待审核商品数量 |
| `seckillNum` | 可参与秒杀 | 可参与秒杀活动数量 |
| `waitPayBill` | 未对账结算单 | 未对账结算单数量 |
| `selfPickNum` | 待自提 | 待自提数量 |
| `alertQuantityNum` | 库存预警 | 库存告警商品/SKU 数量 |

`orderOverview` 字段：

| 字段 | 说明 |
| --- | --- |
| `uvNum` | UV 人次 |
| `orderNum` | 下单数量 |
| `orderMemberNum` | 下单人数 |
| `orderAmount` | 下单金额 |
| `paymentOrderNum` | 付款订单数量 |
| `paymentsNum` | 付款人数 |
| `paymentAmount` | 付款金额 |
| `refundOrderNum` | 退单笔数 |
| `refundOrderPrice` | 退单金额 |
| `orderConversionRate` | 下单转化率 |
| `paymentsConversionRate` | 付款转化率 |
| `overallConversionRate` | 整体转化率 |

`goodsRankList[]` 字段：

| 字段 | 说明 |
| --- | --- |
| `goodsId` | 商品 ID |
| `goodsName` | 商品名称 |
| `num` | 销售数量 |
| `price` | 销售金额 |

截图卡片建议取值：

| 页面模块 | 推荐取值 |
| --- | --- |
| 营业额（元） | `result.storeIndexStatistics.orderPrice`，或业务更偏“付款金额”时取 `result.orderOverview.paymentAmount` |
| 订单总量 | `result.storeIndexStatistics.orderNum`，或本期经营统计取 `result.orderOverview.orderNum` |
| 在线商品（件） | `result.storeIndexStatistics.goodsNum` |
| 库存预警 | `result.storeIndexStatistics.alertQuantityNum` |
| 待发货 | `result.storeIndexStatistics.unDeliveredOrder` |
| 待售后 | `result.storeIndexStatistics.returnGoods + result.storeIndexStatistics.returnMoney` |
| 售后 | 可取售后数量接口的汇总，见下文 |
| 异常包裹 | 当前未看到独立异常包裹聚合字段，前端可先展示 0 或后续新增物流异常统计接口 |

### 8. 工作台快捷入口

截图中的 8 个入口目前更适合前端固定路由配置，后端没有看到专门的“供货工作台快捷入口配置接口”。

建议入口映射：

| 入口 | 目标接口/页面 |
| --- | --- |
| 数据概览 | `GET /store/dashboard/overview` 或统计详情页 |
| 店铺管理 | `GET /store/settings/storeSettings` |
| 发布商品 | `POST /store/goods/goods/create` 对应发布页 |
| 商品管理 | `GET /store/goods/goods/list` |
| 订单管理 | `GET /store/order/order` |

| 资产管理 | `GET /store/dashboard/assets`、结算单 `/store/order/bill/getByPage` |

### 9. 供货商工作台推荐联调顺序

1. 店铺登录：`POST /store/passport/login/userLogin` 或短信登录。
2. 工作台首屏：`GET /store/dashboard/overview`。
3. 资产卡片/资产页：`GET /store/dashboard/assets`。
4. 订单角标：`GET /store/order/order/orderNum`。
5. 售后角标：`GET /store/order/afterSale/afterSaleNumVO`。
6. 公告/违规消息：`GET /store/message/storeMessage?status=UN_READY&pageNumber=1&pageSize=10`。
7. 库存预警列表：`GET /store/goods/goods/list/stock?pageNumber=1&pageSize=10`。
